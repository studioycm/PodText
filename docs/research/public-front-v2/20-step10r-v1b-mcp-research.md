# Step 10R-V1b MCP Research

## Scope

Step 10R-V1b replaces the short public-front icon list with a Heroicon-enum-backed
registry, preserves legacy icon aliases, and introduces one shared lazy searchable icon
picker for public settings icon fields.

## Local Evidence

- Preflight tree was clean.
- `4c545eb feat: add default image fallback settings` is present.
- `docs/phase-02/current-project-state.md`, the central ledger, and the next-sequence
  document agree that Step 10R-V1b is the first pending mini-step.
- `PublicFrontCardTemplateRegistry::icons()` currently returns 17 legacy icon keys.
- `PublicFrontCardIconResolver` maps those legacy keys to
  `Filament\Support\Icons\Heroicon` enum cases.
- Icon settings are currently separate `Select` fields in
  `app/Filament/Pages/PublicContentSettings.php` for card-template parts,
  item-page podcast identity, item-page info fields, item-page date fields, and
  contributor compact-count icon.
- `contributors_page.cards.compact_count_icon` currently uses a separate
  `document-text` one-value registry and should move behind the same V1b icon boundary.

## Laravel Boost Findings

Tools used: `application_info`, `database_schema`, and `search_docs`.

- Boost confirmed Laravel 13.18.0, Filament 5.6.7, Livewire 4.3.3, Pest 4.7.4, and
  local SQLite.
- The schema still stores public settings in the Spatie `settings` table; no new data
  table is required for V1b.
- Filament 5 form `Select` supports `allowHtml()`, `searchable()`,
  `getSearchResultsUsing()`, and `getOptionLabelUsing()`. `getOptionLabelUsing()` is
  required so a selected value can render without preloading every option.
- Filament docs warn that HTML option labels must be generated safely. PodText should
  generate all picker HTML app-side from trusted `Heroicon` enum cases only.
- The docs also support avoiding preloading large option lists by using lazy search
  callbacks instead of `options()`.

## FilamentExamples Findings

Access level: `search_examples` snippet/source access only. No separate
fetch/read/detail tool was exposed.

Query batches:

- `Icon Picker Select Field with Live Icon Display`,
  `select with custom html values and search results`, `heroicon select field preview`
- refined pass: `reusable select form helper allowHtml getSearchResultsUsing`,
  `settings page icon select allowHtml searchable`,
  `form select getSearchResultsUsing getOptionLabelUsing`

Relevant example:

- **Select With Custom HTML Options**
  - Path:
    `v4/forms/select-with-custom-html-values-and-search-results/app/Filament/Resources/Categories/Schemas/CategoryForm.php`
  - Pattern to adapt: `Select::make('icon')->allowHtml()->searchable()` with labels
    generated by `Filament\Support\generate_icon_html($case)`.
  - Pattern to avoid: the example also uses `options()` over all `Heroicon::cases()`;
    V1b requires lazy search results so the full enum is not materialized into the form
    payload.
  - PodText adaptation: wrap the safe HTML label and lazy search callbacks in one
    app-owned `IconSelect` helper, store only normalized finite tokens, and render
    icons exclusively through `PublicFrontCardIconResolver`.

Neighboring useful snippets:

- Searchable `Select` examples using `getSearchResultsUsing()` confirmed the callback
  shape and that options can be resolved lazily.
- Enum icon examples confirmed current Filament code uses
  `Filament\Support\Icons\Heroicon` enum values rather than string icon names.

## Implementation Implications

- Add `PublicFrontIconRegistry` as the one icon-token source of truth.
- Normalize legacy aliases (`calendar`, `podcast`, `arrow_right`, etc.) to Heroicon enum
  case names while keeping aliases permanent for old settings rows and imports.
- Keep a special `none` no-icon token because it is part of the existing 17-key public
  icon vocabulary.
- Add a settings migration to normalize saved nested icon settings to enum-name tokens.
- Replace every public-settings icon `Select` with the shared `IconSelect` helper.
- Tests should assert alias compatibility, invalid fallback, lazy search subsets, and
  bounded settings-page payload behavior.
